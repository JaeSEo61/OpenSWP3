# OpenSWP3

메인 브랜치를 fork해서 메인 브랜치로로 이미 다 받은 상황이라 그대로 진행하였음

3팀은 미니 도구 모음집을 구현  
각 팀원별로 다른 프로그램을 구현하고 각각의 c파일을 구현해서 push

<br>

## 팀원별 임무 분담

단위 변환기: 김동하  
간단 계산기: 남경호  
점수 평균 계산기: 서채은  
문자열 처리: 최재서  
랜덤 비밀번호 생성기: 황병호

<br>

## 기능 파일

각 기능별로 c 코드 파일을 따로 만들어서 각 함수로 구현하였음  
단위 변환기 => conversion.c  
간단 계산기 => pmsd.c  
점수 평균 계산기 => average.c  
문자열 처리 => string_program.c  
랜덤 비밀번호 생성기 => rand_pw.c

<br>

## c파일 기능 별 설명

메인 화면: 번호를 입력하면 해당 프로그램을 실행하도록 설계 및 구현  
conversion.c : 입력 번호에 따라서 단위를 변환하는 프로그램  
pmsd.c: 정수2개와 번호 하나에 따라서 해당 정수 값 2개의 사칙연산을 진행하는 프로그램  
average.c : 정수를 입력 받고 해당 정수들의 평균 값을 계산해주는 프로그램  
string_program.c : 입력된 번호에 따라 다른 문자열 처리를 제공(단순 출력, 문자열 길이, 대소문자 변환)  
rand_pw.c: 랜덤 정수로 비밀번호를 만들어주는 프로그램 제공

<br>

## 병합 충돌 상황

추가로 이 도구들을 실행시키기 위한 메인 출력 화면을 함께 구현  
다만 마지막 메인 화면 구현에 있어서 김동하, 최재서가 동시에 빈 파일인 main.c를 클론하였고,  
이 과정에서 팀원 김동하의 변경 내용을 먼저 merge를 하였고,  
팀원 최재서가 똑같이 push하는 과정에서 최신 버전이 아니였기 때문에 reject 되었고  
최신화를 위해 pull을 하였으나, 기존에 작업해놓은 main.c가 있었기 때문에  
이 과정에서 merge conflict 가 발생하였음

<br>

## 코드 리뷰 후 처리

코드 리뷰를 통해 main.c를 확인해보니  
팀원 김동하가 올린 main.c의 코드에서 한글 부분이 깨져서 보여  
기존 내용을 없애고 팀원 최재서가 작성한 main.c파일로 코드를 대체함

<br>

## 추가 변경점

일부 팀원들이 작성한 c파일 코드 중에서 main.c파일이 따로 존재하고  
이 파일안에 main함수가 존재하는 형태인데,  
c파일안에 함수 이름을 다르게 설정한게 아니라, main으로 설정해서  
후에 컴파일 시 오류가 발생할 수 있음을 인지

오류 발생을 막기 위해 일부 c파일 내부의 함수이름을  
main에서 각 프로그램의 특징을 말해주는 함수이름으로 변경하였음

<br>

## 실행 방법
코드 파일을 전부 visual studio 프로젝트에 집어넣고 컴파일 하게 되면 메인 화면이 출력됨
그 후에 실행시키고자 하는 프로그램에 맞는 번호를 입력하고 해당 프로그램을 실행하면 됨
